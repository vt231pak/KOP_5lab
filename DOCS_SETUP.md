# 🚀 Як запустити локальну документацію TypeDoc

## Опція 1️⃣: Через Python (найпростіше)

### Windows / Mac / Linux:

```bash
# Перейдіть в корень проекту
cd "c:/Users/tosha/OneDrive/Рабочий стол/KOP_5lab"

# Запустіть Python сервер
python -m http.server 8000
```

### Відкрийте в браузері:
```
http://localhost:8000/docs/index.html
```

---

## Опція 2️⃣: Через Node.js (http-server)

### Встановіть http-server:
```bash
npm install -g http-server
```

### Запустіть сервер:
```bash
http-server ./docs
```

### Відкрийте в браузері:
```
http://localhost:8080
```

---

## Опція 3️⃣: Через VS Code Live Server (найкраще для редагування)

### 1. Встановіть розширення:
- Відкрийте VS Code
- Перейдіть до Extensions (Ctrl+Shift+X)
- Пошукайте "Live Server"
- Встановіть від Ritwick Dey

### 2. Відкрийте docs/index.html:
```bash
code docs/index.html
```

### 3. Натисніть правою кнопкою на файлі:
```
"Open with Live Server"
```

### Браузер автоматично відкриється з документацією!

---

## Якщо документація не згенерована - генеруйте її:

```bash
npm run docs
```

Це створить/оновить папку `docs/` з HTML документацією.

---

## Що ви побачите в документації:

1. **Ліве меню** - всі компоненти та модулі
2. **Card компонент** - зі всіма JSDoc коментарями
3. **GameGrid компонент** - з описом комплексності
4. **Все решта компонентів** - GameStatsDisplay, DifficultyButton тощо
5. **Типи та інтерфейси** - CardProps, GameGridProps
6. **Приклади використання** - код приклади для кожного компоненту

---

## 🎥 ДЛЯ ВІДЕО:

1. Запустіть `npm run docs` в терміналі
2. Покажіть генерацію в консолі
3. Відкрийте `http://localhost:8000/docs/index.html` або натисніть на `docs/index.html`
4. Прокрутіть документацію
5. Натисніть на компоненти (Card, GameGrid)
6. Покажіть JSDoc коментарі та типи

Це буде іспанувачне доказ, що документація працює локально! ✅
